# 개요

이 섹션에서는 자신만의 검증자를 운영하는 방법을 안내합니다. 검증자 운영은 `story` 합의 클라이언트를 통해 수행할 수 있습니다.

<Note>
  아래 작업들은 노드를 실행할 필요가 없습니다! 하지만 스테이킹 보상에 참여하고 싶다면 검증자 노드를 실행해야 합니다.
</Note>

진행하기 전에 위임자와 검증자의 차이점을 이해하는 것이 중요합니다:

* A **validator**는 서명된 키가 `priv_validator_key.json` 파일 아래의 `story` 데이터 디렉토리에 있는 합의에 참여하는 전체 노드입니다. 검증자 키 세부 정보를 출력하려면 [검증자 키 내보내기 섹션](/network/become-a-validator#validator-key-export)
* A **delegator**는 `IP`를 보유하고 직접 검증자를 운영할 필요 없이 합의 보상에 참여하고자 하는 계정 운영자를 의미합니다.

귀하의 `story` 바이너리가 있는 동일한 폴더에 `.env` 파일을 추가하고 `PRIVATE_KEY` 계정에 `IP` 자금이 있어야 합니다. **모든 아래 작업에 위임자 계정을 사용하는 것을 권장합니다.**

<Note>
  검증자 자체로도 트랜잭션을 발행할 수 있습니다. 검증자에 해당하는 EVM
  개인 키를 얻으려면 [검증자
  키 내보내기](#validator-key-export) 섹션을 참조하세요.
</Note>

The `.env` 파일은 다음과 같아야 합니다 *(0x 접두사를 추가하지 마세요):*

```bash
# ~/.env
PRIVATE_KEY=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

이제 다양한 검증자 작업을 수행할 준비가 되었습니다! 아래에서 CLI를 통해 지원되는 모든 작업을 안내해드리겠습니다:

## 검증자 키 내보내기

기본적으로 `./story init`를 실행하면 검증자 키가 생성됩니다. 검증자 키를 보려면 다음 명령을 실행하세요:

```bash
./story validator export [flags]
```

이것은 압축 및 비압축 형식으로 검증자 공개 키 파일을 출력합니다. 기본적으로 공개 식별을 위해 16진수로 인코딩된 압축 키를 사용합니다.

```text
Compressed Public Key (hex): 03bdc7b8940babe9226d52d7fa299a1faf3d64a82f809889256c8f146958a63984
Compressed Public Key (base64): A73HuJQLq+kibVLX+imaH689ZKgvgJiJJWyPFGlYpjmE
Uncompressed Public Key (hex): 04bdc7b8940babe9226d52d7fa299a1faf3d64a82f809889256c8f146958a6398496b9e2af0a3a1d199c3cc1d09ee899336a530c185df6b46a9735b25e79a493af
EVM Address: 0x9EacBe2C3B1eb0a9FC14106d97bd3A1F89efdDCc
Validator Address: storyvaloper1p470h0jtph4n5hztallp8vznq8ehylsw9vpddx
Delegator Address: story1p470h0jtph4n5hztallp8vznq8ehylswtr4vxd
```

**사용 가능한 플래그:**

* `--export-evm-key`: (문자열) 검증자의 파생된 EVM 개인 키를 기본 데이터 구성 디렉토리로 내보냅니다
* `--export-evm-key-path`: (문자열) 검증자의 파생된 EVM 개인 키에 대한 다른 다운로드 위치를 지정합니다
* `--keyfile`: (문자열) Tendermint 키 파일 경로 (기본값 "/home/ubuntu/.story/story/config/priv\_validator\_key.json")

<Tip>
  검증자로서 트랜잭션을 발행하고 싶고 위임자로서가 아니라면, 키를 `.env` 파일로 내보내고 IP가 전송되었는지 확인하세요. 예를 들어 `./story validator export --export-evm-key --evm-key-path
    .env`
</Tip>

## 검증자 생성

새 검증자를 생성하려면 다음 명령을 실행하세요:

```bash Locked token node
./story validator create
			 --stake ${AMOUNT_TO_STAKE_IN_WEI} \
			 --moniker ${VALIDATOR_NAME} \
       --rpc ${rpc} \
		   --chain-id ${chain_id} \
			 --commission-rate ${rate} \
 			 --unlocked=false
```

```Text Unlocked token node
./story validator create
			 --stake ${AMOUNT_TO_STAKE_IN_WEI} \
			 --moniker ${VALIDATOR_NAME} \
       --rpc ${rpc} \
		   --chain-id ${chain_id} \
			 --commission-rate ${rate} \
```

이것은 `priv_validator_key.json`에 저장된 검증자 키에 해당하는 검증자를 생성하고, `{$AMOUNT_TO_STAKE_IN_WEI}` IP를 자체 스테이킹합니다.

<Note>
  합의에 참여하려면 최소 1024 IP가 스테이킹되어야 합니다 (`1024000000000000000000 wei`와 동일)!
</Note>

아래는 검증자 설정을 추가로 사용자 정의할 수 있는 선택적 플래그 목록입니다:

**사용 가능한 플래그:**

* `--stake`: 검증자가 자체 위임할 금액을 wei 단위로 설정합니다 (기본값은 `1024000000000000000000` wei).
* `--moniker`: 네트워크의 사용자들에게 보이는 검증자의 사용자 정의 이름을 정의합니다.
* `--chain-id`: 트랜잭션의 Chain ID를 지정합니다. 기본값은 `1516`입니다.
* `--commission-rate`: 검증자의 수수료율을 bips로 설정합니다 (1% = 100 bips). 예를 들어, `1000`는 10% 수수료를 나타냅니다 (기본값은 `1000`).
* `--explorer`: 블록체인 탐색기의 URL을 지정합니다 (기본값: [https://www.storyscan.io](https://www.storyscan.io)).
* `--keyfile`: Tendermint 키 파일의 경로를 지정합니다 (기본값: `$HOME/.story/story/config/priv_validator_key.json`).
* `--max-commission-change-rate`: 검증자의 수수료가 변경될 수 있는 최대 비율을 bips로 설정합니다. 예를 들어, `100`는 최대 1% 변경을 나타냅니다 (기본값은 `1000`).
* `--max-commission-rate`: 검증자가 부과할 수 있는 최대 수수료율을 bips로 정의합니다. 예를 들어, `5000`는 50% 최대 비율을 허용합니다 (기본값은 `5000`).
* `--private-key`: 트랜잭션 서명에 지정된 개인 키를 사용합니다. 설정하지 않으면 `priv_validator_key.json` 사용될 것입니다.
* `--rpc`: 네트워크에 연결할 RPC URL을 설정합니다 (기본값: [https://mainnet.storyrpc.io](https://mainnet.storyrpc.io)).
* `--unlocked`: 잠금 해제된 토큰 스테이킹이 지원되는지 결정합니다 (`true` 잠금 해제된 스테이킹의 경우, `false` 잠긴 스테이킹의 경우). 기본값은 `true`입니다.
* `--story-api`: 잠재적인 자금 손실을 방지합니다. 기본적으로 `http://localhost:1317`값을 설정해야 합니다

### 생성 명령어 사용 예시

```bash
story validator create
	--stake 1024000000000000000000
  --moniker "timtimtim"
  --commission-rate 700
  --validator-pubkey "<validator_pubkey>" # if you dont have a .env
  --rpc "https://mainnet.storyrpc.io"
	--chain-id 1514
```

### 검증자 확인하기

생성이 완료되면 `Explorer URL`를 사용하여 트랜잭션을 확인하세요. 성공적이라면, 검증자 공개키(*에서 찾을 수 있는`priv_validator_key.json` 파일*)가 다음 엔드포인트에 나열된 것을 확인할 수 있습니다:

```bash
curl https://testnet.storyrpc.io/validators | jq .
```

축하합니다, 이제 Story의 최초 IP 검증자 중 한 명이 되셨습니다!

## 검증자 스테이킹

기존 검증자에게 스테이킹하려면 다음 명령어를 실행하세요:

```bash
./story validator stake \
   --validator-pubkey ${VALIDATOR_PUB_KEY_IN_HEX} \
   --stake ${AMOUNT_TO_STAKE_IN_WEI}
   --staking-period ${STAKING_PERIOD}
```

* 귀하의 `${VALIDATOR_PUB_KEY_IN_HEX}`는 `./story validator export` 명령어를 실행하여 `Compressed Public Key (hex)`에서 찾을 수 있습니다.
* 트랜잭션이 유효하려면 최소 1024 IP 상당(`*1024000000000000000000 wei`)을 스테이킹해야 합니다

스테이킹이 완료되면 `Explorer URL`를 사용하여 트랜잭션을 확인하세요. 앞서 언급했듯이, [검증자 엔드포인트](https://mainnet.storyrpc.io/validators)를 사용하여 검증자의 새로운 투표력을 확인할 수 있습니다.

**사용 가능한 플래그:**

* `--validator-pubkey`: (문자열) 스테이킹할 검증자의 공개키
* `--stake`: (문자열) wei 단위로 스테이킹할 IP 양
* `--chain-id`: (정수) 트랜잭션에 사용할 체인 ID (기본값: 1514)
* `--explorer`: (문자열) 블록체인 탐색기 URL
* `--help`, `-h`: 스테이크 명령어에 대한 도움말 표시
* `--private-key`: (문자열) 트랜잭션에 사용할 개인키
* `--rpc`: (문자열) 네트워크에 연결할 RPC URL
* `--staking-period`: (스테이킹 기간) 스테이킹 기간 (옵션: "flexible", "short", "medium", "long") (기본값: flexible)
* `--story-api`: 잠재적인 자금 손실을 방지합니다. 기본적으로 `http://localhost:1317`값을 설정해야 합니다

### 스테이킹 명령어 사용 예시

```bash
./story validator stake \
  --validator-pubkey 03bdc7b8940babe9226d52d7fa299a1faf3d64a82f809889256c8f146958a63984 \
  --stake 1024000000000000000000
  --staking-period "short"
```

## 검증자 언스테이킹

검증자로부터 언스테이킹하려면 다음 명령어를 실행하세요:

```bash
./story validator unstake \
  --validator-pubkey ${VALIDATOR_PUB_KEY_IN_HEX} \
  --unstake ${AMOUNT_TO_UNSTAKE_IN_WEI} \
	--delegation-id ${ID_STAKING_PERIOD}
```

이것은 `${AMOUNT_TO_UNSTAKE_IN_WEI}` IP를 선택한 검증자로부터 언스테이킹합니다. 트랜잭션이 유효하려면 최소 1024 IP 상당(`*1024000000000000000000 wei`)을 언스테이킹해야 합니다.

스테이킹 작업과 마찬가지로 `Explorer URL`를 사용하여 트랜잭션을 확인하고 [검증자 엔드포인트](https://mainnet.storyrpc.io/validators)를 통해 검증자의 새로 감소된 투표력을 다시 확인하세요.

**사용 가능한 플래그:**

* `--chain-id`: (정수) 트랜잭션에 사용할 체인 ID (기본값: 1514)
* `--delegation-id`: (uint32) 위임 ID (유연한 스테이킹의 경우 0)
* `--explorer`: (문자열) 블록체인 탐색기 URL (기본값: "[https://www.storyscan.io](https://www.storyscan.io)")
* `--help`, `-h`: 언스테이크 명령어에 대한 도움말
* `--private-key`: (문자열) 트랜잭션에 사용할 개인키
* `--rpc`: (문자열) 네트워크에 연결할 RPC URL (기본값: "[https://mainnet.storyrpc.io](https://mainnet.storyrpc.io)")
* `--unstake`: (문자열) wei 단위로 언스테이크할 양
* `--validator-pubkey`: (문자열) 검증자의 16진수로 인코딩된 33바이트 secp256k1 공개키
* `--story-api`: 잠재적인 자금 손실을 방지합니다. 기본적으로 `http://localhost:1317`값을 설정해야 합니다

### 언스테이킹 명령어 사용 예시

```bash
./story validator unstake \
   --validator-pubkey 03bdc7b8940babe9226d52d7fa299a1faf3d64a82f809889256c8f146958a63984 \
   --unstake 1024000000000000000000 \
   --delegation-id 1
```

## 검증자 대리 스테이킹

다른 위임자를 대신하여 스테이킹하려면 다음 명령어를 실행하세요:

```bash
./story validator stake-on-behalf \
  --delegator-address ${DELEGATOR_EVM} \
  --validator-pubkey ${VALIDATOR_PUB_KEY_IN_HEX} \
  --stake ${AMOUNT_TO_STAKE_IN_WEI} \
  --staking-period ${STAKING_PERIOD} \
  --rpc
  --chain-id
```

이것은 `${AMOUNT_TO_STAKE_IN_WEI}` IP를 제공된 위임자를 대신하여 검증자에게 스테이킹합니다. 트랜잭션이 유효하려면 최소 1024 IP 상당(`*1024000000000000000000 wei`)을 스테이킹해야 합니다.

다른 스테이킹 작업과 마찬가지로 `Explorer URL`를 사용하여 트랜잭션을 확인하고 [검증자 엔드포인트](https://mainnet.storyrpc.io/validators)를 통해 검증자의 증가된 투표력을 다시 확인하세요.

**사용 가능한 플래그:**

* `--chain-id`: (정수) 트랜잭션에 사용할 체인 ID (기본값: 1514)
* `--delegator-address`: (문자열) 위임자의 EVM 주소
* `--explorer`: (string) 블록체인 익스플로러의 URL (기본값: "[https://www.storyscan.io](https://www.storyscan.io)")
* `--help`, `-h`: stake-on-behalf 명령어에 대한 도움말
* `--private-key`: (string) 트랜잭션에 사용되는 개인키
* `--rpc`: (string) 네트워크에 연결할 RPC URL (기본값: "[https://mainnet.storyrpc.io](https://mainnet.storyrpc.io)")
* `--stake`: (string) 검증인이 자체 위임할 wei 단위의 금액
* `--staking-period`: (stakingPeriod) 스테이킹 기간 (옵션: "flexible", "short", "medium", "long") (기본값: flexible)
* `--validator-pubkey`: (string) 검증인의 16진수로 인코딩된 33바이트 secp256k1 공개키
* `--story-api`: 잠재적인 자금 손실 방지. 기본적으로 `http://localhost:1317`를 값으로 설정해야 합니다

### Stake-on-behalf 명령어 사용 예시

```bash
./story validator stake-on-behalf \
   --delegator-address 0xF84ce113FCEe12d78Eb41590c273498157c91520 \
   --validator-pubkey 03e42b4d778cda2f3612c85161ba7c0aad1550a872f3279d99e028a1dfa7854930 \
   --stake 1024000000000000000000 \
   --staking-period "short" \
	 --rpc \
   --chain-id
```

## 검증인 Unstake-on-behalf

위임자를 대신하여 언스테이킹을 할 수도 있습니다. 하지만 이를 위해서는 해당 위임자의 승인된 운영자로 등록되어 있어야 합니다. 운영자로서 다른 위임자를 대신하여 언스테이킹하려면 다음 명령어를 실행하세요:

```bash
./story validator unstake-on-behalf \
  --delegator-address ${DELEGATOR_PUB_KEY_IN_HEX} \
  --validator-pubkey ${VALIDATOR_PUB_KEY_IN_HEX} \
  --unstake ${AMOUNT_TO_STAKE_IN_WEI} \
  --rpc \
  --chain-id
```

이것은 `${AMOUNT_TO_STAKE_IN_WEI}` IP를 위임자를 대신하여 검증인으로부터 언스테이킹합니다. 이는 귀하가 해당 위임자의 등록된 운영자라는 가정 하에 가능합니다. 트랜잭션이 유효하려면 최소 1024 IP 가치(`*1024000000000000000000 wei`)를 언스테이킹해야 합니다.

다른 스테이킹 작업과 마찬가지로 `Explorer URL`를 사용하여 트랜잭션을 확인하고 우리의 [검증인 엔드포인트](https://mainnet.storyrpc.io/validators)를 통해 검증인의 감소된 투표력을 다시 확인하세요.

**사용 가능한 플래그:**

* `--chain-id`: (int) 트랜잭션에 사용할 체인 ID (기본값: 1514)
* `--delegator-address`: (string) 위임자의 EVM 주소
* `--explorer`: (string) 블록체인 익스플로러의 URL (기본값: "[https://www.storyscan.io](https://www.storyscan.io)")
* `--help`, `-h`: unstake-on-behalf 명령어에 대한 도움말
* `--private-key`: (string) 트랜잭션에 사용되는 개인키
* `--rpc`: (string) 네트워크에 연결할 RPC URL (기본값: "[https://mainnet.storyrpc.io](https://mainnet.storyrpc.io)")
* `--unstake`: (string) wei 단위로 언스테이킹할 금액

  --redelegate ${AMOUNT_TO_REDELEGATE_IN_WEI}

  --chain-id

* `--private-key`: (string) 트랜잭션에 사용되는 개인키

## 운영자 설정

위임자는 자신을 대신하여 언스테이킹하거나 재위임할 운영자를 추가할 수 있습니다. 운영자를 추가하려면 다음 명령어를 실행하세요:

* `--chain-id` int 트랜잭션에 사용할 체인 ID (기본값 1514)
* `--explorer` string 블록체인 탐색기의 URL (기본값 "[https://www.storyscan.io](https://www.storyscan.io)")
* `--operator` string 위임자에 대한 운영자 설정
* `--private-key` string 트랜잭션에 사용되는 개인키
* `--rpc` string 네트워크에 연결할 RPC URL (기본값 "[https://mainnet.storyrpc.io](https://mainnet.storyrpc.io)")

```bash
./story validator set-operator \
  --operator ${OPERATOR_EVM_ADDRESS} \
  --rpc \
  --chain-id \
  --story-api ${STORY_API_URL}
```

트랜잭션이 유효하려면 트랜잭션을 제출하는 지갑에 최소 1 IP가 필요합니다.

## 검증인을 다른 기기로 마이그레이션

<Warning>
  검증인 노드를 새로운 기기로 마이그레이션하기 전에, 현재 노드가 완전히 종료되었는지 확인하세요. 활성 검증인을 복원하려고 시도하면 "이중 서명"이라는 중대한 오류가 발생할 수 있으며, 이는 위임된 지분이 삭감되는 결과를 초래할 수 있습니다.
</Warning>

1. 검증인을 위한 새로운 환경을 구성하는 것부터 시작하세요. 새로운 전체 노드가 네트워크의 최신 블록과 완전히 동기화되었는지 확인하세요.
2. 실수로 인한 이중 서명을 방지하기 위해, 새로운 인스턴스를 활성화하기 전에 원래 검증인 노드를 완전히 종료하는 것이 필수적입니다. 시스템 재부팅 후 자동으로 재시작되는 것을 방지하기 위해 Story 서비스 파일을 삭제하는 것을 권장합니다. 또한 `priv_validator_key.json`와 `priv_validator_state.json`를 백업하고 활성 검증인을 실행 중인 현재 서버에서 제거하세요. 이러한 단계를 건너뛰면 블록 누락이나 다른 페널티가 발생할 수 있습니다.

```bash
# Step 1: Stop the original validator node
sudo systemctl stop <your_service_file_name>.service

# Step 2: Disable the Story service to prevent automatic restarts
sudo systemctl disable <your_service_file_name>.service

# Step 3: Delete the Story service file to prevent it from starting on reboot
sudo rm /etc/systemd/system/<your_service_file_name>.service

# Step 4: Back up the `priv_validator_key.json` file securely, e.g., using SFTP:
# Use an SFTP client or a secure method to download the file without displaying it in the terminal
# If needed for verification purposes only, you may view it with the following command:
cat ~/.story/story/config/priv_validator_key.json

# Step 5: Remove the `priv_validator_key.json` file from the current server
rm ~/.story/story/config/priv_validator_key.json
```

3. 다음 위치로 이동 `priv_validator_key.json`와 `priv_validator_state.json` 내의 `~/.story/story/config/` 디렉토리에 있는 새 머신에서. 이 파일을 이전 검증인의 백업 복사본으로 교체하세요.

<Warning>
  진행하기 전에, 원래 서버에서 이전 검증인을 종료하고 다시 시작하지 마세요!
</Warning>

4. 개인 키 파일을 전송한 후, 새로운 설정에서 검증인 노드를 다시 시작하세요. 이렇게 하면 검증인이 네트워크에 다시 통합되어 검증 역할을 재개할 수 있습니다.
