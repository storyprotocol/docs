Story의 다양한 레지스트리는 프로토콜의 전역 상태를 위한 주요 디렉토리/저장소로 기능합니다. 당연히 해당 저장소를 업데이트하는 기능도 포함하고 있습니다.

특정 IP의 상태를 관리하는 [⚙️ IP Accounts](/concepts/ip-asset/ip-account)와 달리, **registry**는 프로토콜의 더 넓은 상태를 감독합니다.

# 레지스트리 유형

다음은 Story의 모든 레지스트리입니다.

## [IP Asset Registry](/concepts/registry/ip-asset-registry)

프로토콜에 IP를 등록하는 역할을 담당합니다.

## [Group IP Asset Registry](/concepts/registry/group-ip-asset-registry)

Group IP Asset의 등록과 유지를 담당합니다.

## [License Registry](/concepts/registry/license-registry)

IP Asset에 라이선스 조건을 첨부하고, 파생작을 등록하고, 새로운 라이선스 템플릿을 만드는 등 프로토콜 내의 모든 라이선스 관련 상태를 저장합니다.

## [Module Registry](/concepts/registry/module-registry)

Story에 자유롭게 등록된 모듈과 훅의 전역 목록을 유지하고 업데이트합니다
