Royalty Module은 Story에서 IP 간의 수익 흐름을 정의합니다. 더 구체적으로는, 부모와 자식 [🧩 IP Assets](/concepts/ip-asset/overview) 간의 수익 흐름을 정의합니다. 수익 흐름이 발생하는 두 가지 일반적인 시나리오가 있습니다:

1. 라이선스 발행 - [License Token](/concepts/licensing-module/license-token)을 발행할 때 `mintingFee`. 누군가가 이를 지불할 때(파생작을 등록하거나 단순히 라이선스를 보유하고자 하는 경우), 수익은 조상 체인을 따라 흐르게 됩니다.
2. 직접 팁 전송 - 누군가가 IP에 직접 수익을 보내면, 이는 체인을 따라 흐르게 됩니다.

## 상위 수준 예시

아래 예시는 ([Liquid Absolute Percentage](/concepts/royalty-module/liquid-absolute-percentage) 사용) IP Asset 4(IPA4)가 IPA3에 1,000,000 WIP을 팁으로 보낼 때 발생하는 상황을 보여줍니다.

1. 수익은 먼저 Royalty Module 컨트랙트로 흐릅니다
2. Royalty Module은 **royalty stack** (15%)
3. LAP는 IPA3의 수익에서 일정 수익을 받기로 한 라이선스 계약을 맺은 추가 조상들에게 자금을 분배합니다.

## 로열티 정책

로열티 정책은 두 IP Asset 간의 라이선스 계약의 구성 요소입니다. 이는 수익 흐름이 실제로 어떻게 이루어지는지를 정의합니다.

Royalty Module은 우리 팀이 직접 만든 화이트리스트/네이티브 정책과 사용자가 만든 외부 정책을 모두 지원합니다.

<Note>
  부모가 없는 IP Asset은 다양한 로열티 정책으로 라이선스를 발행할 수 있는 반면, 파생 IP Asset은 부모의 로열티 정책을 상속받습니다.

  또한, IP Asset이 각각의 파생작과 맺는 모든 연결에는 항상 하나의 로열티 정책이 적용됩니다.
</Note>

### 화이트리스트/네이티브 로열티 정책

이러한 정책들은 거버넌스 화이트리스팅이 필요하며 조상들에게 로열티 토큰 분배를 보장합니다.

1. [Liquid Absolute Percentage (LAP)](/concepts/royalty-module/liquid-absolute-percentage)
2. [Liquid Relative Percentage (LRP)](/concepts/royalty-module/liquid-relative-percentage)

### 외부 로열티 정책

이러한 정책들은 허가 없이 등록될 수 있으며 자체적인 로열티 및 수익 분배 규칙을 규정합니다.

* [외부 로열티 정책](/concepts/royalty-module/external-royalty-policies)

## 로열티 토큰 % vs 로열티 스택 %

파생작을 만들 때, 창작자는 "내 IP 수익 중 얼마를 내가 가지고 얼마를 조상 IP들에게 줄 것인가?"라는 질문에 답해야 합니다.

이 질문에 답하기 위해서는 두 가지 개념이 중요합니다:

1. 로열티 토큰 - 각 IP Asset에는 100,000,000개의 로열티 토큰이 연결되어 있으며, 각 토큰은 IP 로열티 금고에 들어오는 자본의 0.000001%를 나타냅니다. 이러한 로열티 토큰의 보유자들은 관련 IP 로열티 금고에 있는 수익 토큰을 청구할 수 있습니다.
2. 로열티 스택 - 화이트리스트/네이티브 로열티 정책을 통해 조상들에게 지불해야 하는 IP 수익의 비율입니다. 외부 로열티 정책은 로열티 스택 비율을 사용하지 않습니다 - 화이트리스트/네이티브 로열티 정책만 사용합니다.

아래 시나리오를 상상해보세요:

* IP1은 루트 IP Asset입니다.
* IP2는 IP1의 파생작입니다.
* 사용자 A는 IP1의 로열티 토큰 100%를 보유
* 사용자 B는 IP2의 로열티 토큰 20%를 보유
* 사용자 C는 IP2의 로열티 토큰 80%를 보유
* IP2의 로열티 스택은 10%입니다 - 이는 네이티브/화이트리스트 정책을 통한 모든 조상 IP들이 파생작을 만들기 위해 IP2의 수익 10%를 지불해야 함을 의미합니다. 이 경우에는 IP1이라는 하나의 조상만 있으며, IP1은 파생작을 만들기 위해 IP2의 미래 수익의 10%를 요구합니다.

아래 이미지는 IP2에 백만 WIP이 지불된 예시를 보여줍니다. 이미지에서는 지불이 이루어질 때 전체 파생 체인의 각 로열티 토큰 보유자가 받는 금액을 확인할 수 있습니다.

<Frame>
  <img src="/images/concepts/rt-vs-rs.png" alt="Royalty Token Distribution Example" />
</Frame>

* RT 보유자 A - 백만 WIP 지불에서 100k WIP를 받습니다. 로열티 스택 비율이 먼저 지불되고 RT 보유자 A는 IP1의 로열티 토큰 100%를 보유하고 있어서 전체 100k WIP를 가져갑니다.
* RT 보유자 B - 백만 WIP 지불에서 180k WIP를 받습니다. IP2 보유자들은 전체적으로 원래 백만 WIP 지불에서 900k WIP를 받습니다. 이 900k WIP는 B와 C인 IP2의 다양한 로열티 토큰 보유자들 사이에서 나눠집니다. B는 IP2의 로열티 토큰 20%를 보유하고 있어서 900k WIP \* 20% = 180k를 받습니다.
* RT 보유자 C - 백만 WIP 지불에서 720k WIP를 받습니다. IP2 보유자들은 전체적으로 원래 백만 WIP 지불에서 900k WIP를 받습니다. 이 900k WIP는 B와 C인 IP2의 다양한 로열티 토큰 보유자들 사이에서 나눠집니다. C는 IP2의 로열티 토큰 80%를 보유하고 있어서 900k WIP \* 80% = 720k를 받습니다.

## 파생 체인 구성

<Frame>
  <img src="/images/concepts/derivative-chain-config.png" alt="Derivative Chain Configurations" />
</Frame>

파생 체인은 여러 가지 구성을 가질 수 있습니다.

각 IP 자산은 총 로열티 100%로 제한됩니다. 조상에 대해 100% 이상의 로열티 토큰을 예약하게 되는 라이선스를 발행할 때는 이는 의미가 없으므로 되돌려집니다.
