지적 재산권은 [🧩 IP 자산](/concepts/ip-asset) 및 관련된 [⚙️ IP 계정](/concepts/ip-asset/ip-account)으로 표현되며, 이는 각 IP의 핵심 신원으로 설계된 스마트 컨트랙트입니다. 또한 우리는 다양한 [🧱 모듈](/concepts/modules)을 통해 IP 자산에 파생물 생성, IP 분쟁, 수익 자동 분배 등의 기능을 추가할 수 있습니다.

위 다이어그램에서 언급된 계층들을 간단히 소개하겠습니다:

## [🧩 IP 자산](/concepts/ip-asset)

IP를 온체인으로 가져오고 싶을 때, ERC-721 NFT를 발행합니다. 이 NFT는 당신의 IP에 대한 **소유권**을 나타냅니다.

그런 다음, **등록**을 통해 NFT를 우리 프로토콜의 [IP 자산 레지스트리](/concepts/registry/ip-asset-registry)에 등록합니다. 이는 [⚙️ IP 계정](/concepts/ip-asset/ip-account)을 배포하여 "IP 자산"을 생성합니다. 해당 컨트랙트의 주소가 IP 자산의 식별자가 됩니다 (`ipId`).

기본 NFT는 다른 NFT처럼 거래/판매될 수 있으며, 새로운 소유자는 IP 자산과 관련된 모든 수익을 소유하게 됩니다.

## [⚙️ IP 계정](/concepts/ip-asset/ip-account)

IP 계정은 IP 자산과 연결된 스마트 컨트랙트로, 두 가지 주요 기능을 수행합니다:

1. IP로부터 생성된 라이선스와 로열티 등 관련 IP 자산의 데이터 저장
2. 다양한 모듈에 의한 이 데이터의 활용을 촉진. 예를 들어, 라이선싱, 수익/로열티 공유, 리믹스 및 기타 중요한 기능들은 IP 계정의 프로그래밍 가능성 덕분에 가능합니다.

IP 계정의 주소가 IP 자산의 식별자입니다 (`ipId`).

## [🧱 모듈](/concepts/modules)

모듈은 IP 계정의 기능을 정의하고 확장하는 맞춤형 스마트 컨트랙트입니다. 모듈은 개발자들이 각 IP에 대한 기능과 상호작용을 만들어 IP를 진정으로 프로그래밍 가능하게 만듭니다.

우리는 이미 몇 가지 핵심 모듈을 보유하고 있습니다:

1. [📜 라이선싱 모듈](/concepts/licensing-module): IP 간의 부모\<->자식 관계를 생성하여, 라이선스 조건(저작권 표시 필수, 10% 수익 공유 등)에 의해 제한되는 IP의 파생물을 가능하게 함
2. [💸 로열티 모듈](/concepts/royalty-module): 라이선스 조건에서 협상된 수익 공유에 따라 IP 간의 수익 흐름을 자동화
3. [❌ 분쟁 모듈](/concepts/dispute-module): IP의 분쟁 및 신고를 처리
4. [👥 그룹핑 모듈](/concepts/grouping-module): IP들을 그룹으로 묶을 수 있게 함
5. [👀 메타데이터 모듈](/concepts/metadata-module): IP 자산의 메타데이터 관리 및 조회

## [🗂️ 레지스트리](/concepts/registry)

우리 프로토콜의 다양한 레지스트리는 프로토콜의 전역 상태를 위한 주요 디렉토리/저장소 역할을 합니다. 특정 IP의 상태를 관리하는 IP 계정과 달리, 레지스트리는 프로토콜의 더 넓은 상태를 감독합니다.

## [💊 프로그래머블 IP 라이선스 (PIL)](/concepts/programmable-ip-license)

PIL은 IP 자산이 법적으로 라이선스될 수 있는 특정 **라이선스 조건**을 정의하는 실제 오프체인 법적 계약입니다. 예를 들어, IP 자산이 어떻게 상업화되고, 리믹스되거나, 귀속되는지, 그리고 누가 어떤 조건 하에서 그것을 할 수 있는지를 정의합니다.

우리는 이러한 조건들을 온체인에 매핑하여 다른 사람들이 귀하의 IP를 원활하고 투명하게 라이선스할 수 있도록 IP 자산에 조건을 쉽게 첨부할 수 있게 했습니다.
