하나의 [🧩 IP Asset](/concepts/ip-asset/overview)이 등록되면, 연결된 **IP Account**가 부여됩니다. IP Account는 수정된 ERC-6551(토큰 바운드 계정) 구현입니다. 이는 Story의 모듈과의 상호작용 권한을 제어하거나 IP의 관련 데이터를 저장하기 위해 IP Asset에 바인딩된 별도의 컨트랙트입니다. 등록 시, IP Asset에는 고유 ID가 할당됩니다. 이 ID는 IP Asset에 바인딩된 IP Account의 주소입니다.

<img src="/images/concepts/ip-account.png" alt="IP Account Diagram" />

IP Account는 주로 두 가지 역할을 합니다:

1. IP에서 생성된 라이선스 토큰이나 로열티 토큰과 같은 관련 자산의 메타데이터와 소유권 세부 정보를 포함한 포괄적인 IP 관련 데이터를 저장합니다.
2. 다양한 모듈에서 이 데이터의 활용을 용이하게 합니다. 이러한 모듈들은 IP Account와 상호작용하고 데이터를 생성하고 저장합니다. 예를 들어, 라이선싱, 수익/로열티 공유, 리믹싱, IP 분쟁 및 기타 모듈들은 IP Account의 프로그래밍 가능성 덕분에 가능해집니다.

<Note>
  기본 NFT가 이전되면, 새로운 소유자는 자동으로 연관된 IP Asset과 IP Account의 소유자가 됩니다.
</Note>

## `execute`와 `executeWithSig`

IP Account의 주요 기능은 일반적인 `execute()` 함수로, 인코딩된 바이트 데이터를 통해 Story 내의 임의의 모듈을 호출할 수 있게 합니다(따라서 향후 모듈을 위해 확장 가능). 또한, `executeWithSig()` 함수가 있어 사용자가 트랜잭션에 서명하고 다른 사람이 원활한 UX를 위해 대신 실행할 수 있게 합니다.
