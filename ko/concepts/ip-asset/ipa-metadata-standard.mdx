이것은 IP Asset과 연관된 JSON 메타데이터로, IP Account 내부에 저장됩니다. 메타데이터를 설정하기 위해서는`setMetadata(...)`를 IP Account 내에서 호출해야 하며, 그 다음`metadata()`를 호출하여 읽을 수 있습니다.

## 속성 및 구조

아래는 IP 메타데이터에서 제공해야 할 중요한 속성들입니다. **필수 항목** 열은 특정 필드가 어디에 필요한지를 나타냅니다:

* 🔍 Story Explorer - 이 필드는 Story Explorer에서 IP를 표시하는 데 도움이 됩니다
* 🕵️ [상업적 침해 검사](/concepts/story-attestation-service) - 이 필드는 IP가 **commercial**인 경우(즉, `commercialUse = true` 라이선스 조건이 첨부된 경우)에 필요합니다. 이러한 필드들을 사용하여 IP에 대한 침해 검사를 실행할 것입니다.
  * 참조: [현재 제한사항](/concepts/story-attestation-service#current-limitations).
* 🤖 AI Agents - AI Agents와 관련된 메타데이터 표시에 사용됨

### 콘텐츠 해싱

콘텐츠를 해싱하기 위해 `imageHash` 또는 `mediaHash` 필드에는 SHA-256 해싱 알고리즘을 사용할 수 있습니다. 다음은 JavaScript에서 이를 수행하는 방법의 예시입니다:

<CodeGroup>
  ```typescript TypeScript
  import { toHex, Hex } from "viem";

  // get hash from a file
  async function getFileHash(file: File): Promise<Hex> {
    const arrayBuffer = await file.arrayBuffer();
    const hashBuffer = await crypto.subtle.digest("SHA-256", arrayBuffer);
    return toHex(new Uint8Array(hashBuffer), { size: 32 });
  }

  // get hash from a url
  async function getHashFromUrl(url: string): Promise<Hex> {
    const response = await axios.get(url, { responseType: "arraybuffer" });
    const buffer = Buffer.from(response.data);
    return "0x" + createHash("sha256").update(buffer).digest("hex");
  }
  ```

  ```shell Shell
  shasum -a 256 myfile.jpg
  ```
</CodeGroup>

<Warning>

### 관계 타입

다음은 `relationships` 속성에 사용할 수 있는 다양한 관계 타입입니다.

#### 스토리 관계

1. **APPEARS\_IN** - 캐릭터가 챕터에 등장함.

2. **BELONGS\_TO** - 챕터가 책에 속함.

3. **PART\_OF** - 책이 시리즈의 일부임.

4. **CONTINUES\_FROM** - 챕터가 이전 챕터에서 이어짐.

5. **LEADS\_TO** - 하나의 사건이 결과로 이어집니다.

6. **FORESHADOWS** - 하나의 사건이 미래의 발전을 암시합니다.

7. **CONFLICTS\_WITH** - 한 캐릭터가 다른 캐릭터와 갈등합니다.

8. **RESULTS\_IN** - 결정이 중요한 변화를 초래합니다.

9. **DEPENDS\_ON** - 서브플롯이 메인 플롯에 의존합니다.

10. **SETS\_UP** - 프롤로그가 이야기를 설정합니다.

11. **FOLLOWS\_FROM** - 한 챕터가 이전 챕터를 따릅니다.

12. **REVEALS\_THAT** - 반전이 예상치 못한 일이 발생했음을 드러냅니다.

13. **DEVELOPS\_OVER** - 캐릭터가 이야기 과정에서 발전합니다.

14. **INTRODUCES** - 챕터가 새로운 캐릭터나 요소를 소개합니다.

15. **RESOLVES\_IN** - 갈등이 특정 결과로 해결됩니다.

16. **CONNECTS\_TO** - 주제가 주요 서사와 연결됩니다.

17. **RELATES\_TO** - 서브플롯이 중심 주제와 관련됩니다.

18. **TRANSITIONS\_FROM** - 장면이 한 배경에서 다른 배경으로 전환됩니다.

19. **INTERACTED\_WITH** - 캐릭터가 다른 캐릭터와 상호작용했습니다.

20. **LEADS\_INTO** - 사건이 절정으로 이어집니다.?\
    **PARALLEL - story** 병렬로 또는 같은 시간대에 일어나는 일

#### AI 관계

1. **TRAINED\_ON** - 모델이 데이터셋으로 학습됩니다.

2. **FINETUNED\_FROM** - 모델이 기본 모델에서 미세조정됩니다.

3. **GENERATED\_FROM** - 이미지가 미세조정된 모델에서 생성됩니다.

4. **REQUIRES\_DATA** - 모델이 학습을 위한 데이터를 필요로 합니다.

5. **BASED\_ON** - 리믹스가 특정 워크플로우를 기반으로 합니다.

6. **INFLUENCES** - 샘플 데이터가 모델 출력에 영향을 미칩니다.

7. **CREATES** - 파이프라인이 미세조정된 모델을 생성합니다.

8. **UTILIZES** - 워크플로우가 기본 모델을 활용합니다.

9. **DERIVED\_FROM** - 미세조정된 모델이 기본 모델에서 파생됩니다.

10. **PRODUCES** - 모델이 생성된 이미지를 만듭니다.

11. **MODIFIES** - 리믹스가 기본 워크플로우를 수정합니다.

12. **REFERENCES** - AI 생성 이미지가 원본 데이터를 참조합니다.

13. **OPTIMIZED\_BY** - 모델이 특정 알고리즘에 의해 최적화됩니다.
