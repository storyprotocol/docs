License Template은 IP에 대한 다양한 라이선스 조건을 정의하는 코드로 작성된("프로그래밍 가능한") 법적 프레임워크입니다. 예를 들면:

* "상업적 사용이 허용되나요?" - true/false (bool)
* "라이선스가 양도 가능한가요?" - true/false (bool)
* "상업적 사용의 경우, 받는 로열티 비율은 얼마인가요?" - number

이러한 조건과 값들은 License Template마다 다릅니다.

첫 번째(그리고 현재 유일한) License Template의 예시는 Story 팀이 직접 개발했으며, Programmable IP License (PIL :pill:)라고 불립니다.

## License Template 요구사항

License Template은 다음을 담당합니다:

* 모든 매개변수, 가능한 값, 그리고 해당하는 법적 문구가 포함된 실제 오프체인 법적 계약 템플릿에 대한 링크를 제공`licenseTextUrl`.
  * 라이선스 프레임워크가 Story와 호환되려면 법적 문구는**반드시** 명확하고 매개변수화되어 있어야 하며, 각 라이선스 매개변수는 각 값의 가능한 결과를 설정해야 합니다.
  * 각 License Template의 매개변수 값("License Template 조건"이라고 함)은 각 라이선스 계약의 법적 문구를 결정합니다.
* 정의하기`struct` License Terms 구조체(아래 설명)에 인코딩되어야 하는 매개변수의 특정 정의와 함께.
* License Terms에 대한 등록 방법과 getter 제공.
* **확인**하기**minter**와 주소**가 파생물을 연결할 때 License Template 조건에 따라 해당 작업을 수행할 수 있는지**.
  * 이러한 조건들은 License Template 자체나 훅을 통해 강제될 수 있습니다. 파생물 생성의 제한, LNFT 보유자에 대한 토큰 게이팅, 라이선스 제공자의 창작 통제, KYC 등이 포함될 수 있습니다. 이는 각 License Template의 구현에 달려있습니다.
* **파생물이 여러 부모를 가지고 있거나 가질 예정인 경우 License Terms가 호환되는지 확인**

## 자체 템플릿 만들기

자체 License Template(PIL과 같은)을 만들 수 있지만, 프로토콜에 완전히 통합되려면 Story 팀의 승인을 받아야 합니다.
