---
title: License Token 발행하기
description: TypeScript에서 IP Asset으로부터 License Token을 발행하는 방법을 알아보세요.
---

이 섹션에서는 [License Token](/concepts/licensing-module/license-token)을 [IP Asset](/concepts/ip-asset)으로부터 발행하는 방법을 보여줍니다. IP Asset에 [License Terms](/concepts/licensing-module/license-terms)가 첨부된 경우에만 License Token을 발행할 수 있습니다. License Token은 ERC-721로 발행됩니다.

License Token을 발행하는 데에는 두 가지 이유가 있습니다:

1. 라이선스를 보유하고 라이선스에 명시된 대로 기본 IP Asset을 사용할 수 있습니다(예: "적절한 귀속을 제공하고 수익의 5%를 공유하는 한 상업적으로 사용할 수 있음")
2. 라이선스 토큰을 사용하여 다른 IP Asset을 파생물로 연결합니다.*단, 나중에 보시겠지만, 일부 SDK 함수는 파생물을 등록하기 위해 명시적으로 라이선스 토큰을 먼저 발행할 필요가 없으며, 실제로 이를 백그라운드에서 처리합니다.*

### 전제 조건

튜토리얼을 시작하기 전에 완료해야 할 몇 가지 단계가 있습니다.

1. 완료하기 [TypeScript SDK Setup](/developers/typescript-sdk/setup)
2. License Terms가 추가된 IP Asset. IPA에 License Terms를 추가하는 방법 알아보기 [here](/developers/typescript-sdk/attach-terms).

## 1. 라이선스 발행

IP Asset (`ipId = 0x01`)에 License Terms (`licenseTermdId = 10`)가 첨부되어 있다고 가정해 보겠습니다. 우리는 이러한 조건으로 특정 지갑 주소(`0x02`)에 2개의 License Token을 발행하고 싶습니다.

<Warning>
  일부 IP Asset에는 라이선스를 발행하는 사용자가 `defaultMintingFee`를 지불해야 하는 라이선스 조건이 첨부되어 있을 수 있습니다. 이에 대한 예시는 [TypeScript Tutorial](https://github.com/storyprotocol/typescript-tutorial/blob/main/scripts/derivative/registerDerivativeCommercialCustom.ts)에서 확인할 수 있습니다.
</Warning>

<Note>
  라이선스 토큰은 `licenseTermsId`가 이미 IP Asset에 첨부되어 있어 공개적으로 사용 가능한 라이선스인 경우에만 발행할 수 있습니다. 그러나 IP 소유자는 [private license](/concepts/licensing-module/license-token#private-licenses)를 IP Asset에 첨부되지 않은 `licenseTermsId`로 라이선스 토큰을 발행할 수 있습니다.
</Note>

<Info>
  관련 문서:
  [license.mintLicenseTokens](/sdk-reference/license#mintlicensetokens)
</Info>

```typescript main.ts
// you should already have a client set up (prerequisite)
import { client } from "./client";

async function main() {
  const response = await client.license.mintLicenseTokens({
    licenseTermsId: "10",
    licensorIpId: "0x641E638e8FCA4d4844F509630B34c9D524d40BE5",
    receiver: "0x641E638e8FCA4d4844F509630B34c9D524d40BE5", // optional. if not provided, it will go to the tx sender
    amount: 2,
    maxMintingFee: BigInt(0), // disabled
    maxRevenueShare: 100, // default
    txOptions: { waitForTransaction: true },
  });

  console.log(
    `License Token minted at transaction hash ${response.txHash}, License IDs: ${response.licenseTokenIds}`
  );
}

main();
```

### 1a. License Token 발행에 대한 제한 설정

이는 라이선스 토큰의 발행 대상이나 방법에 대한 제한을 설정하고자 하는 IP Asset 소유자를 위한 참고 사항입니다. 다음과 같은 설정이 가능합니다:

* 발행할 수 있는 라이선스의 최대 수량 설정
* 발행자나 발행 수량에 따른 동적 수수료 부과
* 토큰 발행이 허용된 지갑 화이트리스트 설정

... 등이 있습니다. 자세한 내용은 다음을 확인하세요 [라이선스 설정](/concepts/licensing-module/license-config) 섹션을 참조하세요.

## 2. 파생작 등록하기

이제 라이선스 토큰을 발행했으니, 이를 보유하거나 IP 자산을 파생작으로 연결하는 데 사용할 수 있습니다. 다음 페이지에서 자세히 알아보겠습니다.

*나중에 보시겠지만, 일부 SDK 함수는 파생작을 등록하기 위해 명시적으로 라이선스 토큰을 먼저 발행할 필요가 없으며, 실제로 백그라운드에서 이를 처리합니다.*

### 2a. 라이선스 토큰이 필요하지 않다면 왜 사용하나요?

다음과 같은 경우에 **필요합니다** 파생작을 등록하기 위한 라이선스 토큰이:

* 라이선스 토큰에 비공개 라이선스 조항이 포함되어 있어서, 소유자가 수동으로 발행한 라이선스 토큰이 있는 경우에만 파생작으로 등록할 수 있습니다. 자세한 내용은 [here](/concepts/licensing-module/license-token#private-licenses).
* 라이선스 토큰(NFT)은 `mintingFee` 발행 비용이 들지만, 마켓플레이스에서 더 저렴한 가격에 구매할 수 있었습니다. 이 경우 더 비싼 `defaultMintingFee`.
