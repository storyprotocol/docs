이 튜토리얼에서는 TypeScript SDK를 사용하여 IP 자산에 돈("팁")을 보내는 방법을 배우게 됩니다.

## 설명

이 시나리오에서, Mickey Mouse를 나타내는 부모 IP 자산이 있다고 가정해봅시다. 다른 누군가가 그 Mickey Mouse에 모자를 그려서 파생(또는 "자식") IP 자산으로 등록했습니다. 라이선스 조건에 따르면 자식은 모든 상업적 수익의 50%를 부모와 공유해야 합니다(`commercialRevShare = 50`). 다른 누군가(제3자 사용자)가 와서 정말 멋지다고 생각해서 파생작에 2 \$WIP를 보내고 싶어합니다.

이 예제에서는 자식이 이미 파생 IP 자산으로 등록되어 있다고 가정하겠습니다. 이것을 배우는 데 도움이 필요하다면 다음을 확인하세요: [파생작 등록하기](/developers/typescript-sdk/register-derivative).

* 부모 IP ID: `0x42595dA29B541770D9F9f298a014bF912655E183`
* 자식 IP ID: `0xeaa4Eed346373805B377F5a4fe1daeFeFB3D182a`

## 0. 시작하기 전에

튜토리얼을 시작하기 전에 완료해야 할 몇 가지 단계가 있습니다.

1. 설치해야 할 것: [Node.js](https://nodejs.org/en/download) 와 [npm](https://docs.npmjs.com/downloading-and-installing-node-js-and-npm). 이전에 코딩을 해보셨다면 이미 설치되어 있을 것입니다.
2. Story Network 테스트넷 지갑의 개인 키를 다음에 추가하세요: `.env` file:

```text env
WALLET_PRIVATE_KEY=<YOUR_WALLET_PRIVATE_KEY>
```

3. 선호하는 RPC URL을 `.env` 파일에 추가하세요. 우리가 제공하는 공개 기본값을 사용하면 됩니다:

```text env
RPC_PROVIDER_URL=https://aeneid.storyrpc.io
```

4. 의존성 설치하기:

```bash Terminal
npm install @story-protocol/core-sdk viem
```

## 1. Story Config 설정하기

다음 `utils.ts` 파일에 Story Config를 설정하기 위한 다음 코드를 추가하세요:

* 관련 문서: [TypeScript SDK 설정](/developers/typescript-sdk/setup)

```typescript utils.ts
import { StoryClient, StoryConfig } from "@story-protocol/core-sdk";
import { http } from "viem";
import { privateKeyToAccount, Address, Account } from "viem/accounts";

const privateKey: Address = `0x${process.env.WALLET_PRIVATE_KEY}`;
export const account: Account = privateKeyToAccount(privateKey);

const config: StoryConfig = {
  account: account,
  transport: http(process.env.RPC_PROVIDER_URL),
  chainId: "aeneid",
};
export const client = StoryClient.newClient(config);
```

## 2. 파생 IP 자산에 팁 주기

이제 `main.ts` 파일을 만듭니다. 우리는 `payRoyaltyOnBehalf` 함수를 사용하여 파생 자산에 지불할 것입니다.

IP 자산에 \[$WIP](https://aeneid.storyscan.io/address/0x1514000000000000000000000000000000000000). **Note that if you don't have enough $WIP로 지불하게 되며, 함수가 자동으로 동등한 양의 $IP into $WIP로 래핑해줄 것입니다.\*\* 둘 중 하나가 충분하지 않으면 실패하게 됩니다.

<Note>
  **화이트리스트에 등록된 수익 토큰**

  우리 프로토콜에서 화이트리스트에 등록된 토큰만 지불("수익") 토큰으로 사용할 수 있습니다. \$WIP는 그 중 하나입니다. 해당 목록을 보려면 [여기](/developers/deployed-smart-contracts)를 확인하세요.
</Note>

이제 우리는 `payRoyaltyOnBehalf` 함수를 호출할 수 있습니다. 이 경우:

1. `receiverIpId` 는 `ipId` 파생(자식) 자산의 것입니다
2. `payerIpId` 는 `zeroAddress` 입니다. 왜냐하면 지불자는 제3자(모자를 쓴 Mickey Mouse가 멋지다고 생각하는 사람)이며, 반드시 다른 IP 자산일 필요는 없기 때문입니다
3. `token` 는 \$WIP의 주소이며, [여기](/concepts/royalty-module/ip-royalty-vault#whitelisted-revenue-tokens)
4. `amount` 는 2입니다. 팁을 주는 사람이 2 \$WIP를 보내고 싶어하기 때문입니다

```typescript main.ts
import { client } from "./utils";
import { zeroAddress, parseEther } from "viem";
import { WIP_TOKEN_ADDRESS } from "@story-protocol/core-sdk";

async function main() {
  const response = await client.royalty.payRoyaltyOnBehalf({
    receiverIpId: "0xeaa4Eed346373805B377F5a4fe1daeFeFB3D182a",
    payerIpId: zeroAddress,
    token: WIP_TOKEN_ADDRESS,
    amount: parseEther("2"), // 2 $WIP
    txOptions: { waitForTransaction: true },
  });
  console.log(`Paid royalty at transaction hash ${response.txHash}`);
}

main();
```

## 3. 자식이 받을 수익 청구하기

이 시점에서 우리는 이미 튜토리얼을 완료했습니다: IP 자산에 팁을 주는 방법을 배웠습니다. 하지만 자식과 부모가 그들의 수익을 청구하고 싶다면 어떻게 할까요?

자식은 2 \$WIP를 받았습니다. 하지만 기억하세요, 라이선스 조건의 `commercialRevenue = 50` 때문에 수익의 50%를 부모 IP와 공유합니다.

자식 IP는 다음을 호출하여 1 \$WIP를 청구할 수 있습니다 `claimAllRevenue` function:

* `ancestorIpId` 는 `ipId` 로열티 금고에 자금이 있는 IP 자산의 것입니다(더 간단히 말하면, 이것은 단순히 자식의 `ipId`입니다)
* `currencyTokens` 는 \$WIP의 주소를 포함하는 배열이며, [여기](/concepts/royalty-module/ip-royalty-vault#whitelisted-revenue-tokens)
* `claimer` 는 자식의 [IP 로열티 금고](/concepts/royalty-module/ip-royalty-vault)와 관련된 로열티 토큰을 보유하는 주소입니다. 기본적으로 이것들은 IP 계정에 있으며, 이는 단순히 자식 자산의 `ipId` 입니다

```typescript main.ts
import { client } from "./utils";
import { zeroAddress, parseEther } from "viem";
import { WIP_TOKEN_ADDRESS } from "@story-protocol/core-sdk";

async function main() {
  // previous code here ...
  const response = await client.royalty.claimAllRevenue({
    ancestorIpId: "0xDa03c4B278AD44f5a669e9b73580F91AeDE0E3B2",
    claimer: "0xDa03c4B278AD44f5a669e9b73580F91AeDE0E3B2",
    currencyTokens: [WIP_TOKEN_ADDRESS],
    childIpIds: [],
    royaltyPolicies: [],
  });

  console.log(`Claimed revenue: ${response.claimedTokens}`);
}

main();
```

## 4. 부모가 받을 수익 청구하기

계속해서, 부모도 자신의 수익을 청구할 수 있어야 합니다. 이 예제에서 부모는 1 $WIP since the child earned 2 $WIP와 `commercialRevShare = 50` 라이선스 조건에서.

우리는 `claimAllRevenue` 함수를 사용하여 수익 토큰을 청구할 것입니다.

1. `ancestorIpId`는 `ipId` 부모("조상") 자산의
2. `claimer`는 부모의 로열티 토큰을 보유하고 있는 주소입니다 [IP Royalty Vault](/concepts/royalty-module/ip-royalty-vault). 기본적으로, 이들은 부모 자산의 `ipId`인 IP Account에 있습니다
3. `childIpIds`는 자식 자산의 `ipId`를 가질 것입니다
4. `royaltyPolicies`는 로열티 정책의 주소를 포함할 것입니다. [Royalty Module](/concepts/royalty-module)에서 설명한 대로, 이는 `RoyaltyPolicyLAP` 또는 `RoyaltyPolicyLRP`이며, 이는 라이선스 조건에 따라 다릅니다. 이 경우, `RoyaltyPolicyLAP`를 사용한다고 가정해 보겠습니다. [Deployed Smart Contracts](/developers/deployed-smart-contracts)로 이동하여 올바른 주소를 찾으세요.
5. `currencyTokens`는 \$WIP의 주소를 포함하는 배열입니다. 이는 [here](/concepts/royalty-module/ip-royalty-vault#whitelisted-revenue-tokens)

```typescript main.ts
import { client } from "./utils";
import { zeroAddress, parseEther } from "viem";
import { WIP_TOKEN_ADDRESS } from "@story-protocol/core-sdk";

async function main() {
  // previous code here ...

  const response = await client.royalty.claimAllRevenue({
    ancestorIpId: "0x089d75C9b7E441dA3115AF93FF9A855BDdbfe384",
    claimer: "0x089d75C9b7E441dA3115AF93FF9A855BDdbfe384",
    currencyTokens: [WIP_TOKEN_ADDRESS],
    childIpIds: ["0xDa03c4B278AD44f5a669e9b73580F91AeDE0E3B2"],
    royaltyPolicies: ["0xBe54FB168b3c982b7AaE60dB6CF75Bd8447b390E"],
  });

  console.log(`Claimed revenue: ${response.claimedTokens}`);
}

main();
```
