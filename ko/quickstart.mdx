## 배포된 스마트 컨트랙트

배포된 스마트 컨트랙트의 주소는 [here](/developers/deployed-smart-contracts)에서 확인할 수 있습니다. 두 가지 종류의 컨트랙트가 있습니다:

* [Story Protocol Core](https://github.com/storyprotocol/protocol-core-v1) - 이 저장소는 핵심 프로토콜 로직을 포함하며, 얇은 IP 레지스트리([IP Asset Registry](/concepts/registry/ip-asset-registry)), 일련의 [🧱 Modules](/concepts/modules)로 구성되어 있으며 [📜 Licensing](/concepts/licensing-module), [💸 Royalty](/concepts/royalty-module), [❌ Dispute](/concepts/dispute-module), 메타데이터, 그리고 모듈과 사용자 접근 제어를 관리하는 모듈 관리자에 대한 로직을 정의합니다.
* [Story Protocol Periphery](https://github.com/storyprotocol/protocol-periphery-v1)- 코어 컨트랙트가 기본 프로토콜 로직을 다루는 반면, 주변부 컨트랙트는 UX를 크게 향상시키고 IPA 관리를 단순화하는 프로토콜 확장을 다룹니다. 이는 주로 [📦 SPG](/concepts/spg)를 통해 처리됩니다.

## Story에 IP 등록하기

가장 기본적인 질문부터 시작해보겠습니다:*"내 앱에서 Story에 IP를 등록하려면 어떻게 해야 하나요? 어떻게 하면 되나요?"*

Story에 IP를 등록하려면 먼저 NFT가 필요합니다. 만약 귀하의 IP가 ERC-721 NFT(예: Story의 Azuki나 Pudgy Penguin)라면 이미 준비가 된 것입니다. 그렇지 않다면 오프체인 IP를 대표하는 NFT를 발행해야 합니다. 걱정하지 마세요, 다음 튜토리얼에서 이 과정을 도와드리겠습니다.

그 다음에는 해당 NFT를 Story에 등록하여 최종적으로 [🧩 IP Asset](/concepts/ip-asset)을 생성합니다. "IP Asset"은 Story에 등록된 귀하의 IP로, 다음과 같은 기능이 제공됩니다:

* Story의 모든 [🧱 Modules](/concepts/modules) (투명한 라이선싱, 자동 로열티 지급, 부당하게 등록된 IP에 대한 이의제기 등)
* 다음을 통한 IP 보호: [💊 Programmable IP License (PIL)](/concepts/programmable-ip-license)

### IP 메타데이터와 NFT 메타데이터의 차이점

Story에 IP를 등록하는 개발자들로부터 자주 받는 질문은 다음과 같습니다: *"NFT에는 어떤 메타데이터가 첨부되어야/첨부될 것으로 예상되며, IP Asset에는 별도로 어떤 메타데이터가 필요한가요?"*

이 질문에 대한 답변은 [NFT vs. IP Metadata](/concepts/ip-asset/overview#nft-vs-ip-metadata)를 참조하세요.

## IP 라이선싱

궁금하실 수 있습니다, *"Story의 온체인 라이선싱을 어떻게 활용할 수 있나요? 등록된 IP에 라이선스가 바로 적용되도록 하려면 어떻게 해야 하나요?"*

귀하의 [🧩 IP Asset](/concepts/ip-asset)에 라이선스나 라이선스 조건을 첨부하기 전에, [💊 Programmable IP License (PIL)](/concepts/programmable-ip-license)이 실제로 무엇인지 이해하는 것이 좋습니다. 이 "PIL"은 Story의 [License Terms](/concepts/licensing-module/license-terms)를 정의하며, 이것이 IP Asset에 첨부되면 다른 사람들이 해당 IP Asset을 어떻게 사용할 수 있는지(상업적 사용, 파생물 생성 등)를 정의합니다.

## 로열티 / 수익 공유

이제 궁금하실 수 있습니다, *"내 IP Asset과 다른 사람의 IP Asset 간에 자동 로열티 공유를 어떻게 설정하나요? 그리고 그 지급금은 어떻게 수령하나요?"*

귀하가 [License Terms](/concepts/licensing-module/license-terms)를 [🧩 IP Asset](/concepts/ip-asset)에 첨부할 때, 특정 상업적 조건을 지정할 수 있습니다`commercialRevShare`, 이는 파생 작품이 원본 IP와 공유해야 하는 수익(원본 및 파생 작품의 모든 출처로부터)의 금액입니다. 라이선싱 관련 질문은 위 섹션을 참조하세요.

만약 누군가가 내 IP Asset의 파생물을 만들고 - 라이선스 조건에 `commercialRevShare`가 10%라고 가정했을 때 - 그것으로 수익을 얻는다면, Story는 [💊 Programmable IP License (PIL)](/concepts/programmable-ip-license)을 통해 이 수익의 공유를 강제합니다 ([❌ Dispute Module](/concepts/dispute-module)을 사용한 온체인 분쟁이나 전통적인 법적 중재로 이어질 수 있음) 그리고 프로토콜 수준에서 상위 수익 공유를 처리합니다. 만약 파생 작품이 100 \$WIP를 벌었다면, 내 원본 IP Asset은 10 \$WIP를 청구할 수 있습니다.

## 이의제기

이제 궁금하실 수 있습니다, *"다른 사람이 내 IP를 도용하거나, 사용에 대한 적절한 수익을 지급하지 않을 경우 어떻게 이의를 제기할 수 있나요?"*

"나쁜" IP를 제거하는 두 가지 주요 철학/방법이 있습니다.

첫 번째는 [🕵️ Story Attestation Service](/concepts/story-attestation-service)입니다. 이는 IP 등록 시 자동으로 IP를 검토하는 여러 침해 감지 제공자들로 구성되어 있으며, 그들은 자체적인 방법(AI, 수동 확인 등)을 사용하여 IP가 침해적인 경우(예: 피카츄 사진 등록) 이를 표시합니다. 그러면 [IP Portal](https://portal.story.foundation)과 같은 IP 검색 플랫폼이 이러한 검토 결과를 표시하고 사용자가 해당 IP를 사용할지 여부를 결정할 수 있게 합니다.

*예를 들어, 여러 침해 제공자로부터 수백 개의 표시가 있는 IP는 아마도 정당한 IP가 아닐 것입니다.*

두 번째는 [❌ Dispute Module](/concepts/dispute-module)을 사용하여 프로토콜 수준에서 공식적으로 IP를 태그하고 차단하는 것입니다. 누구나 IP에 표시를 할 수 있으며, 이는 [UMA](https://uma.xyz)와 같은 중재 파트너에게 전달되어 그들이 결정을 내리게 됩니다. 공식적으로 태그되면 해당 IP는 더 이상 프로토콜을 통해 수익을 얻거나 파생물을 만들 수 없습니다.

*예를 들어, 누군가가 IP를 상업적으로 사용하면서 적절한 지불을 하지 않거나, 허용되지 않은 지역에서 사용하거나, NSFW 콘텐츠를 포함하는 경우입니다.*
