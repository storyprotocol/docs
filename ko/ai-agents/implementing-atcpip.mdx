### 계약 조건 생성하기

다음에서 설명된 대로 [백서](https://story.foundation/atcpip), 에이전트들은 요청된 작업에 적절한 계약 조건에 대해 협상할 것입니다:

<Accordion title="Whitepaper Section" icon="circle-info">
  2 **조건 형성**: 제공자 에이전트는 요청을 고려하고 적절한 \
  라이선스 조건을 선택할 것입니다. 사용되는 조건 시스템은 Story의 프로그래밍 가능한 IP 라이선스(PIL)\[6]와 같이 조건의 파싱과 형성을 용이하게 하기 위해 프로그래밍 가능한 특성을 가져야 합니다.

  3 **협상** (선택사항): 에이전트들은 양 당사자에게 적절하다고 판단될 때까지\
  조건이 변경될 수 있는 선택적 협상 단계를 가질 수 있습니다.

  * **반대 조건** (선택사항): 이 단계에서 초기 제안된 조건에 만족하지 않는\
    요청자 에이전트는 반대 제안 조건을 제시할 수 있습니다. 두 에이전트는 표준화된 조건 시스템에 접근할 수 있어 모호함 없이 특정 조항을 참조, 추가 또는 제거할 수 있습니다. 이러한 반대 조건에는 가격, 사용 권한, 기간, 라이선스 제한 또는 기타 협상 가능한 변수에 대한 수정이 포함될 수 있습니다. 일관된 기계 판독 가능한 형식을 사용함으로써 에이전트들은 서로의 제안에 원활하게 반복하고 응답할 수 있어 협상 과정이 논리적으로 일관되고 쉽게 따라갈 수 있도록 보장합니다.
  * **수정된 조건** (선택사항): 상대방의 조건을 받은 후, 제공자 에이전트는 수정된\
    조건을 제시할 수 있으며, 협상 불가능한 핵심 원칙을 유지하면서 요청된 수정사항을 고려합니다. 에이전트들은 구조화된 상호작용을 통해 라이선스 조건을 효과적으로 개선하며, 각 반복은 쟁점 사항을 더 수용 가능한 중간 지점으로 다듬어냅니다. 양측이 동일한 기본 조건 명세를 사용하기 때문에, 이러한 수정은 내부 일관성을 유지하고 시간이 지남에 따라 여러 초안의 비교를 단순화합니다. 이 메커니즘은 두 에이전트가 상호 이해와 상업적 의도를 정확하게 반영하는 합의에 도달할 수 있도록 보장합니다.
  * *이 과정은 합의에 도달할 때까지 여러 번 반복될 수 있습니다*
</Accordion>

에이전트들이 조건에 동의하면, 등록된 자산에 조건을 생성하고 첨부할 수 있습니다:

### License 발행하기

다음에서 언급된 대로 [Whitepaper](https://story.foundation/atcpip), 에이전트들이 조건에 대해 협상을 마치면, 요청자 에이전트는 특정 합의 조건이 첨부된 라이선스를 제공자 에이전트로부터 발행받을 수 있습니다:

<Accordion title="Whitepaper Section" icon="circle-info">
  4 **수락**: 요청자 에이전트는 변경 불가능한 토큰(\
  합의 토큰)을 발행함으로써 조건을 공식적으로 수락합니다. 이 토큰은 제공되는 정보의 사용 방식에 대한 조건과 규칙을 포함합니다. 발행되면 합의는 구속력을 가지며 에이전트는 정보와 관련된 모든 조건을 기억해야 합니다.

  * **지불** (선택사항): 선택된 라이선스 계약 조건에 따라, 일부 에이전트는\
    라이선스를 발행하기 위해 선불 지불을 요구할 수 있습니다. 또한, 조건에 따라 정기적인 수수료나 수익 공유가 필요할 수 있으며, 이는 예를 들어 Story의 로열티 시스템을 통해 자동화될 수 있습니다.
</Accordion>

합의 조건이 IP 자산에 첨부되면, [License Token](/concepts/licensing-module/license-token)을 발행할 수 있습니다:

## MCP와의 통합 예시

우리는 Story의 프로토콜과 상호작용하기 위한 도구를 제공하는 Model Context Protocol (MCP) 서버를 구현했습니다 [the MCP Python SDK](https://github.com/modelcontextprotocol/python-sdk), 그리고 이러한 도구들을 사용하는 AI 에이전트를 만들었습니다.

<CardGroup cols={2}>
  <Card title="Story MCP Server" href="https://github.com/piplabs/story-sdk-mcp" icon="server">
    Agent TCP/IP를 테스트하기 위해 Story의 프로토콜과 상호작용하는 도구가 있는 MCP 서버를 로컬에서 실행하세요.
  </Card>

  <Card title="LangGraph AI Agent" href="https://github.com/sarick-story/langgraph-mcp-agent" icon="robot">
    Story MCP 서버를 사용하여 IP 자산을 생성, 발행 및 등록하는 LangGraph 기반 AI 에이전트입니다.
  </Card>
</CardGroup>

1. Story MCP 서버를 클론하여 IP 발행 + 등록 및 [License Token](/concepts/licensing-module/license-token) 발행과 같은 Story의 프로토콜과 상호작용하는 도구들을 사용해볼 수 있습니다.
2. 그런 다음, 사용자 요청에 따라 이미지를 생성하고, 사용자와 라이선스 조건을 협상하며, Story MCP 서버를 사용하여 Story에서 IP를 발행 + 등록하고 [License Token](/concepts/licensing-module/license-token)을 발행하여 요청 사용자가 작업을 합법적으로 사용할 수 있도록 하는 LangGraph AI 에이전트를 실행하세요.

이론적으로, 에이전트는 에이전트-사용자 설정 대신 에이전트-에이전트 설정에서도 이를 수행할 수 있습니다.

### MCP란 무엇인가요?

> "MCP는 애플리케이션이 LLM에 컨텍스트를 제공하는 방법을 표준화하는 개방형 프로토콜입니다. MCP를 AI 애플리케이션을 위한 USB-C 포트라고 생각해보세요. USB-C가 기기를 다양한 주변기기와 액세서리에 연결하는 표준화된 방법을 제공하는 것처럼, MCP는 AI 모델을 다양한 데이터 소스와 도구에 연결하는 표준화된 방법을 제공합니다."

자세한 내용은 [Model Context Protocol (MCP) website](https://modelcontextprotocol.io/introduction)를 확인하세요.
